# Data Dictionary

## Overview
This dictionary documents the key fields used from raw transactions through customer-level analytics and dashboard output.

| Column Name | Business Meaning | Transformation Logic | Usage in Dashboard |
|---|---|---|---|
| `customer_id` | Unique identifier of each customer | Cast to string, trim whitespace, exclude null/blank for customer-level analysis | Base entity for all KPI denominators and customer-level visuals |
| `invoice_no` | Unique order/invoice reference | Standardized as string; canceled invoices can be flagged (e.g., prefixes) | Distinct count for order frequency |
| `invoice_date` | Transaction timestamp | Parsed to date/datetime format; invalid dates removed | Time-based filtering and year derivation |
| `invoice_year` | Calendar year of invoice | `YEAR(invoice_date)` | Filter (2010/2011 comparison) |
| `quantity` | Units purchased in transaction row | Numeric cast; invalid/negative values reviewed based on business rules | Revenue construction quality check |
| `unit_price` | Selling price per unit | Numeric cast; zero/negative values reviewed/filtered | Revenue construction quality check |
| `revenue` | Transaction-level monetary value | `quantity * unit_price` | Foundation for LTV and aggregated revenue metrics |
| `customer_ltv` | Total value generated by customer | `SUM(revenue)` grouped by `customer_id` | KPI cards, top 20 chart, distribution, scatter color/size |
| `order_frequency` | Number of unique orders per customer | `COUNTD(invoice_no)` grouped by `customer_id` | Scatter x-axis, repeat customer flag logic |
| `avg_order_value` | Average revenue per order per customer | `customer_ltv / order_frequency` (safe divide) | Scatter y-axis and segment interpretation |
| `repeat_customer_flag` | Indicates whether customer has repeat behavior | `CASE WHEN order_frequency > 1 THEN 1 ELSE 0 END` | Repeat customer KPIs and segmentation |
| `ltv_bin` | Discrete customer value band | Bucket `customer_ltv` into fixed intervals (e.g., 0-100, 100-250, etc.) | LTV distribution (binned bar chart) |
| `is_valid_transaction` | Quality indicator for usable rows | `1` for records passing data quality filters | Data quality auditing; optional filter in prep layer |

## KPI Logic Fields

| KPI Field | Formula | Purpose |
|---|---|---|
| `total_customers` | `COUNTD(customer_id)` | Total analyzed customer base |
| `repeat_customers` | `COUNTD(customer_id where repeat_customer_flag = 1)` | Number of repeat customers |
| `repeat_customer_pct` | `repeat_customers / total_customers` | Retention behavior snapshot |
| `average_ltv` | `AVG(customer_ltv)` | Baseline value per customer |
| `average_repeat_ltv` | `AVG(customer_ltv where repeat_customer_flag = 1)` | Value level of retained customers |
